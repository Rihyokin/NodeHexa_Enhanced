# Camera Vision Module Development - 摄像头视觉模块开发文档

> 本文档记录 Hexapod Smart System 摄像头视觉模块的硬件选型、软件架构和开发进展。

---

## 1. 项目概述

### 1.1 目标
为六足机器人集成摄像头视觉系统，实现环境感知、目标追踪和自主导航等智能功能。

### 1.2 主要应用场景

| 场景 | 说明 | 优先级 |
|------|------|--------|
| 目标追踪 | 追踪特定颜色或形状的物体 | 🔴 高 |
| 自主导航 | 基于视觉的避障和路径规划 | 🔴 高 |
| 姿态识别 | 检测机器人自身姿态 | 🟡 中 |
| 环境感知 | 地形识别和高度检测 | 🟡 中 |
| 人机交互 | 手势识别、面部识别 | 🟢 低 |

## 2. 硬件选型

### 2.1 采用方案

**主控制器：OpenMV4 H7 Plus** + **专属舵机拓展板**

| 组件 | 型号 | 说明 |
|------|------|------|
| 主控模块 | OpenMV4 H7 Plus | 高性能视觉处理，内置STM32H743处理器 |
| 舵机拓展板 | 定制版 | 驱动两个云台舵机（水平/垂直） |
| 通信方式 | UART串口 | ESP32与OpenMV交叉通信 |
| 供电 | ESP32 5V → OpenMV VIN | 由ESP32统一供电 |

### 2.2 硬件连接方案

**串口交叉接线：**

```
ESP32          OpenMV4 H7 Plus
───────────────────────────────────
RX (GPIO16)  ↔  TX (P4)
TX (GPIO17)  ↔  RX (P5)
5V          ↔  VIN (5V输入)
GND         ↔  GND
```

**云台舵机连接：**

```
舵机拓展板
    │
    ├── 舵机1 (水平轴) → 云台水平旋转
    └── 舵机2 (垂直轴) → 云台垂直俯仰
```

### 2.4 OpenMV4 H7 Plus 特性

| 参数 | 规格 |
|------|------|
| 处理器 | STM32H743IIK6 (320MHz) |
| RAM | 512KB SRAM + 2MB PSRAM |
| 摄像头 | OV7725 (VGA/QVGA) |
| 存储 | 16MB Flash |
| 接口 | UART, I2C, SPI, USB |
| 支持算法 | 颜色追踪、人脸检测、二维码识别等 |

### 2.3 接口设计

```
            ↓
        图像数据
            ↓

            ↓

```

## 5. 目标追踪实现

### 5.1 系统框图

```
摄像头 → 颜色空间转换 → 颜色阈值分割 → 形态学处理
                                            ↓
                                    轮廓提取与筛选
                                            ↓
                                    目标中心定位
                                            ↓
                                    PID控制输出
                                            ↓
                                    机器人运动控制
```

### 5.2 关键参数

```cpp
// 颜色阈值（可标定）
struct ColorThreshold {
    int hueMin = 0;      // 色调最小值
    int hueMax = 180;     // 色调最大值
    int satMin = 30;     // 饱和度最小
    int satMax = 255;    // 饱和度最大
    int valMin = 30;     // 明度最小
    int valMax = 255;    // 明度最大
};

// PID控制器参数
struct PIDParams {
    float kp = 0.5;      // 比例系数
    float ki = 0.0;      // 积分系数
    float kd = 0.1;      // 微分系数
};
```

### 5.3 状态机

```
IDLE → 检测到目标 → TRACKING → 目标丢失 → SEARCHING → 超时 → IDLE
         ↑                              ↓
         └──────────────────────────────┘
```

## 11. 常见问题与解决方案

### Q1: 图像质量差
**原因**: 光照不足或摄像头参数配置不当
**解决**: 
- 调整曝光时间和增益
- 添加补光灯
- 使用自动白平衡

### Q2: 追踪丢帧
**原因**: 目标移动太快或背景干扰
**解决**:
- 降低机器人移动速度
- 优化颜色阈值
- 添加预测算法

### Q3: ESP32内存不足
**原因**: 图像缓冲区过大
**解决**:
- 降低分辨率
- 使用DMA模式
- 优化内存管理
