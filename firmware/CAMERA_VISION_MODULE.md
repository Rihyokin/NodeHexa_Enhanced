# Camera Vision Module Development - 摄像头视觉模块开发文档

> 本文档记录 Hexapod Smart System 摄像头视觉模块的硬件选型、软件架构和开发进展。

---

## 1. 项目概述

### 1.1 目标
为六足机器人集成摄像头视觉系统，实现环境感知、目标追踪和自主导航等智能功能。

### 1.2 主要应用场景

| 场景 | 说明 | 优先级 |
|------|------|--------|
| 目标追踪 | 追踪特定颜色或形状的物体 | 🔴 高 |
| 自主导航 | 基于视觉的避障和路径规划 | 🔴 高 |
| 姿态识别 | 检测机器人自身姿态 | 🟡 中 |
| 环境感知 | 地形识别和高度检测 | 🟡 中 |
| 人机交互 | 手势识别、面部识别 | 🟢 低 |

## 2. 硬件选型

### 2.1 采用方案

**主控制器：OpenMV4 H7 Plus** + **专属舵机拓展板**

| 组件 | 型号 | 说明 |
|------|------|------|
| 主控模块 | OpenMV4 H7 Plus | 高性能视觉处理，内置STM32H743处理器 |
| 舵机拓展板 | 定制版 | 驱动两个云台舵机（水平/垂直） |
| 通信方式 | UART串口 | ESP32与OpenMV交叉通信 |
| 供电 | ESP32 5V → OpenMV VIN | 由ESP32统一供电 |

### 2.2 硬件连接方案

**串口交叉接线：**

```
ESP32          OpenMV4 H7 Plus
───────────────────────────────────
RX (GPIO16)  ↔  TX (P4)
TX (GPIO17)  ↔  RX (P5)
5V          ↔  VIN (5V输入)
GND         ↔  GND
```

**云台舵机连接：**

```
舵机拓展板
    │
    ├── 舵机1 (水平轴) → 云台水平旋转
    └── 舵机2 (垂直轴) → 云台垂直俯仰
```

## 5. 目标追踪实现（基于OpenMV）

### 5.1 系统框图

```
OpenMV摄像头 → 图像采集 → HSV颜色空间转换 → 颜色阈值分割
                                                ↓
                                        轮廓提取与面积筛选
                                                ↓
                                        目标中心坐标计算
                                                ↓
                                        UART串口发送 → ESP32
                                                ↓
                                        WebSocket转发 → PC显示
```

### 5.2 OpenMV核心代码结构

```python
# main.py (OpenMV端)
import sensor, image, time, pyb

# 初始化摄像头
sensor.reset()
sensor.set_pixformat(sensor.RGB565)
sensor.set_framesize(sensor.QVGA)
sensor.skip_frames(time=2000)

# 初始化串口
uart = pyb.UART(3, 115200, timeout_char=1000)

# 颜色阈值配置 (红色示例)
red_threshold = (30, 100, 15, 127, 15, 127)

while(True):
    img = sensor.snapshot()
    
    # 颜色追踪
    blobs = img.find_blobs([red_threshold])
    
    if blobs:
        largest_blob = max(blobs, key=lambda b: b.area())
        img.draw_rectangle(largest_blob.rect())
        img.draw_cross(largest_blob.cx(), largest_blob.cy())
        
        # 发送目标坐标
        cx = largest_blob.cx()
        cy = largest_blob.cy()
        uart.write(f"TARGET,{cx},{cy}\r\n")
    else:
        uart.write("NO_TARGET\r\n")
    
    time.sleep_ms(50)
```

### 5.3 ESP32串口指令协议

| 指令类型 | 格式 | 说明 |
|----------|------|------|
| 目标数据 | `TARGET,<cx>,<cy>` | OpenMV发送目标中心坐标 |
| 无目标 | `NO_TARGET` | 未检测到目标 |
| 云台控制 | `PAN,<angle>` | 设置水平角度 (0-180) |
| 云台控制 | `TILT,<angle>` | 设置垂直角度 (0-180) |
| 参数配置 | `THRESHOLD,<hmin>,<hmax>,<smin>,<smax>,<vmin>,<vmax>` | 设置颜色阈值 |

### 5.4 关键参数

```cpp
// 颜色阈值（可标定）
struct ColorThreshold {
    int hueMin = 0;      // 色调最小值
    int hueMax = 180;     // 色调最大值
    int satMin = 30;     // 饱和度最小
    int satMax = 255;    // 饱和度最大
    int valMin = 30;     // 明度最小
    int valMax = 255;    // 明度最大
};

// PID控制器参数
struct PIDParams {
    float kp = 0.5;      // 比例系数
    float ki = 0.0;      // 积分系数
    float kd = 0.1;      // 微分系数
};
```

### 5.5 状态机

```
IDLE → 检测到目标 → TRACKING → 目标丢失 → SEARCHING → 超时 → IDLE
         ↑                              ↓
         └──────────────────────────────┘
```

### 6.2 ESP32核心代码示例

```cpp
// openmv_comm.cpp
#include "openmv_comm.h"

void OpenMVComm::begin() {
    serialPort.begin(115200, SERIAL_8N1, 16, 17);
}

void OpenMVComm::parseData() {
    if (serialPort.available() > 0) {
        String data = serialPort.readStringUntil('\n');
        data.trim();
        
        if (data.startsWith("TARGET")) {
            int comma1 = data.indexOf(',');
            int comma2 = data.indexOf(',', comma1 + 1);
            int cx = data.substring(comma1 + 1, comma2).toInt();
            int cy = data.substring(comma2 + 1).toInt();
            
            onTargetDetected(cx, cy);
        } else if (data == "NO_TARGET") {
            onTargetLost();
        }
    }
}

void OpenMVComm::sendCommand(const String& cmd) {
    serialPort.println(cmd);
}
```

## 10. 常见问题与解决方案

### Q1: 图像质量差
**原因**: 光照不足或摄像头参数配置不当
**解决**: 
- 调整曝光时间和增益
- 添加补光灯
- 使用自动白平衡

### Q2: 追踪丢帧
**原因**: 目标移动太快或背景干扰
**解决**:
- 降低机器人移动速度
- 优化颜色阈值
- 添加预测算法

### Q3: ESP32内存不足
**原因**: 图像缓冲区过大
**解决**:
- 降低分辨率
- 使用DMA模式
- 优化内存管理
