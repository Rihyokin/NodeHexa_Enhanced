# Desktop Control Software Development - 桌面控制软件开发文档

> 本文档记录 Hexapod Smart System 桌面端控制软件的开发进展和技术架构。

---

## 1. 项目概述

### 1.1 目标
开发一款游戏风格的桌面端控制软件，以摄像头画面为主界面，通过键盘WASD+QE控制机器人，鼠标控制云台视角，提供沉浸式的控制体验。

### 1.2 主要特性
- 游戏风格UI设计，以摄像头画面为主
- WASD键盘控制机器人移动
- QE键控制机器人旋转
- 鼠标控制云台/摄像头视角
- 实时信息HUD显示（电量、状态等）
- WiFi连接，访问ESP32热点
- 跨平台支持 (Windows/Linux/macOS)
- Python原型 + C++性能版本

## 2. 技术架构

### 2.1 连接方案（基于现有Web架构）

**WiFi连接流程：**
```
桌面应用 → 连接ESP32 WiFi热点 → 访问192.168.4.1 → WebSocket通信
```

**通信架构：**
```
┌─────────────────────────────────────────────────────────────┐
│                    桌面应用                                  │
│  ┌────────────────┐    ┌────────────────┐                 │
│  │   摄像头画面    │    │   控制指令      │                 │
│  │   (MJPEG流)    │ ← │   (WebSocket)   │                 │
│  └────────────────┘    └────────────────┘                 │
└─────────────────────────────────────────────────────────────┘
                         ↓ WiFi (192.168.4.1)
┌─────────────────────────────────────────────────────────────┐
│                     ESP32 主控                             │
│  ┌────────────────┐    ┌────────────────┐                 │
│  │   WiFi热点      │    │   WebSocket    │                 │
│  │   (AP模式)      │ ←→ │   服务器        │                 │
│  └────────────────┘    └────────────────┘                 │
│                         ↓                                   │
│  ┌────────────────┐    ┌────────────────┐                 │
│  │   摄像头数据    │    │   运动控制      │                 │
│  │   (OpenMV)     │    │   (舵机控制)    │                 │
│  └────────────────┘    └────────────────┘                 │
└─────────────────────────────────────────────────────────────┘
```

**关键技术点：**
1. ESP32运行WiFi AP模式，热点名: `HEXAPOD_AP`
2. 桌面应用连接热点后，通过HTTP/WebSocket与ESP32通信
3. 复用现有Web控制界面的通信协议
4. 摄像头画面通过MJPEG流或WebSocket传输

### 2.2 双版本策略

| 版本 | 语言 | 特点 | 适用场景 |
|------|------|------|----------|
| Python版 | Python 3.8+ | 开发快、易调试、功能丰富 | 快速原型、功能验证 |
| C++版 | C++17 | 性能优、响应快、可独立运行 | 产品化、高性能需求 |

### 2.3 依赖库

**Python版本（推荐先行开发）：**
- PyQt5/PySide6 - GUI框架
- websockets - WebSocket客户端
- requests - HTTP请求
- opencv-python - 图像处理（可选）
- numpy - 数据处理

**C++版本（后期优化）：**
- Qt6 - GUI框架
- Qt WebSockets - WebSocket客户端
- Eigen - 数学计算

### 2.4 复用Web控制协议

**参考现有实现：**
- WebSocket端口: `ws://192.168.4.1:80`
- 指令格式: JSON格式
- 运动指令: `{"type":"move","dir":"forward","speed":0.5}`
- 云台指令: `{"type":"gimbal","pan":90,"tilt":45}`
- 摄像头数据: JPEG/MJPEG流

## 5. 通信协议

### 5.1 WebSocket消息格式

**桌面应用 → ESP32：**
```json
{
  "type": "command",
  "action": "move",
  "params": {
    "direction": "forward",
    "speed": 0.5
  }
}
```

**ESP32 → 桌面应用：**
```json
{
  "type": "status",
  "battery": 85,
  "pitch": 0,
  "roll": 0,
  "yaw": 0,
  "speed": "medium"
}
```

### 5.2 指令类型

| 指令 | 格式 | 说明 |
|------|------|------|
| 移动 | `{"type":"move","direction":"forward","speed":0.5}` | 移动方向和速度 |
| 停止 | `{"type":"stop"}` | 立即停止 |
| 云台 | `{"type":"gimbal","pan":90,"tilt":45}` | 云台角度控制 |
| 速度 | `{"type":"speed","level":"fast"}` | 速度等级切换 |
| 状态查询 | `{"type":"query","data":"status"}` | 查询机器人状态 |

### 5.3 摄像头流

**MJPEG流地址：** `http://192.168.4.1:80/mjpeg`

**WebSocket图像传输（备选）：**
```json
{
  "type": "image",
  "data": "base64编码的JPEG数据",
  "timestamp": 1716432000000
}
```
