# Camera Vision Module Development - 摄像头视觉模块开发文档

> 本文档记录 Hexapod Smart System 摄像头视觉模块的硬件选型、软件架构和开发进展。

---

## 1. 项目概述

### 1.1 目标
为六足机器人集成摄像头视觉系统，实现环境感知、目标追踪和自主导航等智能功能。

### 1.2 主要应用场景

| 场景 | 说明 | 优先级 |
|------|------|--------|
| 目标追踪 | 追踪特定颜色或形状的物体 | 🔴 高 |
| 自主导航 | 基于视觉的避障和路径规划 | 🔴 高 |
| 姿态识别 | 检测机器人自身姿态 | 🟡 中 |
| 环境感知 | 地形识别和高度检测 | 🟡 中 |
| 人机交互 | 手势识别、面部识别 | 🟢 低 |

## 2. 硬件选型

### 2.1 摄像头模块对比

| 模块 | 分辨率 | 帧率 | 接口 | 功耗 | 价格 | 推荐度 |
|------|--------|------|------|------|------|--------|
| **OV2640** | 2MP | 60fps | DCMI/I2C | 低 | ¥15 | ⭐⭐⭐⭐⭐ |
| OV5640 | 5MP | 15fps | DCMI/I2C | 中 | ¥25 | ⭐⭐⭐⭐ |
| ESP32-CAM | 2MP | 30fps | SPI/UART | 中 | ¥20 | ⭐⭐⭐⭐ |
| Raspberry Pi Camera | 8MP | 90fps | CSI | 高 | ¥120 | ⭐⭐⭐ |

### 2.2 推荐方案

**方案1：ESP32-CAM（推荐入门）**
```
优点：
- 成本低，资料丰富
- 内置WiFi，可无线传输
- 与主控平台一致（ESP32）

缺点：
- 图像处理能力有限
- 不支持高级算法
```

**方案2：OV2640 + 主控分离**
```
优点：
- 图像质量好
- 可对接高性能处理器
- 灵活性高

缺点：
- 硬件复杂度增加
- 需要额外接线
```

### 2.3 接口设计

```
ESP32主控 ←→ I2C/SPI ←→ 摄像头模块
            ↓
        图像数据
            ↓
      边缘计算处理
            ↓
        控制决策
```

## 5. 目标追踪实现

### 5.1 系统框图

```
摄像头 → 颜色空间转换 → 颜色阈值分割 → 形态学处理
                                            ↓
                                    轮廓提取与筛选
                                            ↓
                                    目标中心定位
                                            ↓
                                    PID控制输出
                                            ↓
                                    机器人运动控制
```

### 5.2 关键参数

```cpp
// 颜色阈值（可标定）
struct ColorThreshold {
    int hueMin = 0;      // 色调最小值
    int hueMax = 180;     // 色调最大值
    int satMin = 30;     // 饱和度最小
    int satMax = 255;    // 饱和度最大
    int valMin = 30;     // 明度最小
    int valMax = 255;    // 明度最大
};

// PID控制器参数
struct PIDParams {
    float kp = 0.5;      // 比例系数
    float ki = 0.0;      // 积分系数
    float kd = 0.1;      // 微分系数
};
```

### 5.3 状态机

```
IDLE → 检测到目标 → TRACKING → 目标丢失 → SEARCHING → 超时 → IDLE
         ↑                              ↓
         └──────────────────────────────┘
```

## 11. 常见问题与解决方案

### Q1: 图像质量差
**原因**: 光照不足或摄像头参数配置不当
**解决**: 
- 调整曝光时间和增益
- 添加补光灯
- 使用自动白平衡

### Q2: 追踪丢帧
**原因**: 目标移动太快或背景干扰
**解决**:
- 降低机器人移动速度
- 优化颜色阈值
- 添加预测算法

### Q3: ESP32内存不足
**原因**: 图像缓冲区过大
**解决**:
- 降低分辨率
- 使用DMA模式
- 优化内存管理
